# GraphNode 기능 명세서 (Functional Specification) - 상세 보강본

본 문서는 **GraphNode** 서비스의 각 기능별 전제 조건, 사용 방법(프로세스), 입력 정보 및 내부 로직을 PM 및 외부 인원이 즉시 파악할 수 있도록 상세히 기술한 문서입니다.

---

## 1. 개요 (Overview)
GraphNode는 사용자의 비정형 데이터(대화, 메모)를 AI가 분석하여 지식 간의 관계를 도출하고, 이를 시각화된 지식 그래프로 변환하여 관리하는 지능형 노트 서비스입니다.

---

## 2. 홈 뷰 (Home View)

### 2.1 웰컴 섹션 및 퀵 챗 (Quick Chat)
1. **기능 개요**
   - 서비스 접속 시 가장 먼저 마주하는 화면으로, 별도의 페이지 이동 없이 즉시 AI와 대화를 시작하거나 파일을 분석할 수 있도록 지원한다.
2. **사용자 이용 프로세스**
   - 홈 중앙의 입력창(ChatSendBox)에 질문이나 요청 사항을 입력한다.
   - 필요 시 파일을 드래그 앤 드롭하거나 클립 아이콘을 클릭하여 첨부한다.
   - 전송 버튼(또는 Enter)을 클릭하면 해당 맥락을 유지한 채 **채팅 뷰**로 자동 이동하며 답변이 시작된다.
3. **데이터 입력 정보**
   - **필수 정보**: 텍스트 메시지 (질문 또는 지시어).
   - **선택 정보**: 이미지 파일, 문서 파일 (PDF, Markdown, Text 등).
   - **설정 항목**: 사용할 LLM 모델 (GPT-4o, Gemini 등).

### 2.2 최근 활동 (Recent Activities)
1. **기능 개요**
   - 사용자가 마지막으로 편집한 노트나 최근 AI와 나눴던 대화 목록을 시각적으로 제공하여 작업의 연속성을 보장한다.
2. **전제 조건**
   - 최소 1개 이상의 저장된 노트 또는 생성된 채팅 스레드가 있어야 목록이 노출된다. (없을 경우 기본 가이드 노출)
3. **데이터 정보**
   - **노트**: 제목(Markdown 첫 줄), 본문 요약, 최종 수정일.
   - **채팅**: AI가 자동 요약한 스레드 제목, 마지막 대화 시간.

---

## 3. 채팅 뷰 (Chat View)

### 3.1 대화 인터페이스 및 히스토리 관리
1. **기능 개요**
   - 특정 주제에 대해 AI와 심층적인 대화를 나누고 정보를 축적한다. 생성된 대화는 지식 그래프의 소스 데이터가 된다.
2. **사용자 이용 프로세스**
   - 좌측 채팅 리스트에서 기존 대화를 선택하거나 '새 대화' 버튼(단축키)을 눌러 시작한다.
   - 실시간으로 생성되는 AI 답변을 확인하며 추가 질문을 이어간다.
   - 답변 내의 마크다운 서식(코드, 표 등)을 활용해 정보를 확인한다.
3. **데이터 입력 정보**
   - **입력 채널**: 텍스트 입력, 파일 업로드 (멀티모달).
   - **파라미터**: 대화 모델(Model) 선택, 시스템 프롬프트(에이전트 설정 기반).

---

## 4. 지식 시각화 뷰 (Visualization View) - Macro & Micro

### 4.1 Macro 시각화 (Macro Graph)
1. **기능 개요**
   - 서비스 전체에 축적된 모든 지식(노트, 채팅)의 거시적 연결 관계를 네트워크 구조로 시각화한다.
2. **핵심 인터랙션 기능**
   - **2D/3D 전환**: 입체적인 공간 탐색(3D)과 평면적인 정돈된 탐색(2D) 모드 지원.
   - **줌 & 팬 (Zoom/Pan)**: 마우스 휠로 확대/축소, 드래그를 통해 광활한 지식 맵을 자유롭게 항해.
   - **노드/엣지 위치 조작**: 개별 노드의 위치를 드래그하여 레이아웃을 임시로 조정 가능.
   - **포커싱 및 줌인**: 특정 지식군 클릭 시 해당 위치로 부드럽게 화면 이동 및 확대.
3. **사용자 이용 프로세스**
   - 상단 토글로 차원(2D/3D)을 선택하고, 사이드바의 클러스터 리스트를 통해 관심 주제로 이동한다.
   - "Update Graph" 버튼을 통해 최신 대화/노트 내용이 반영된 최신 그래프로 갱신한다.

### 4.2 Micro 시각화 (Micro Graph)
1. **기능 개요**
   - 개별 문서(논문, 긴 메모)나 특정 주제 내의 세부 요소(문제점, 방법론, 결과 등) 간의 미시적 관계를 시각화한다.
2. **세부 기능 및 모드**
   - **Network 모드**: 요소 간의 순수한 관계형 구조를 force-directed 레이아웃으로 표현.
   - **Cluster 모드**: 중심이 되는 대상(예: Paper)을 가운데 두고 유형별(Method, Metric 등)로 그룹화하여 배치.
   - **가시성 제어**: 선택된 요소와 직접 연결된 관계(Edge)만 강조하여 복잡도를 낮추는 기능.
   - **속성 범례 (Legend)**: 요소 타입별(Paper, Problem, Result 등) 색상 및 약어 표시, 통계 정보 제공.
3. **사용자 이용 프로세스**
   - 특정 문서를 정밀 분석할 때 화면에 로드되며, 범례를 통해 각 요소의 성격(예: 빨간색은 Problem)을 파악한다.
   - 요소를 클릭하여 해당 항목의 상세 설명 및 근거 문장을 사이드바에서 확인한다.

### 4.3 분석 사이드바 (Insights)
1. **기능 개요**
   - 그래프 형태만으로 파악하기 힘든 지식의 패턴과 통계적 정보를 텍스트와 트리 구조로 제공한다.
2. **구성 요소**
   - **통계/개요**: 데이터 총량, 주요 키워드 클라우드.
   - **계층 트리**: 지식의 위계 구조를 탐록하고 클릭 시 그래프 노드와 연동 포커싱.
   - **패턴 리포트**: AI가 발견한 반복 주제, 생각의 발전 단계, 지식의 공백 등에 대한 분석 리포트.

## 5. 노트 뷰 (Note View)

### 5.1 폴더 및 탐색기
1. **기능 개요**
   - 전통적인 파일 시스템 방식의 문서 관리를 지원한다.
2. **사용자 이용 프로세스**
   - '새 폴더' 버튼으로 카테고리를 생성하고, '새 노트'로 즉시 작성을 시작한다.
   - 드래그 앤 드롭으로 노트를 다른 폴더로 이동시켜 관리한다.

### 5.2 마크다운 실시간 에디터
1. **기능 개요**
   - 텍스트 입력과 동시에 서식이 적용되는 하이브리드 에디터.
2. **동작 및 입력**
   - **실시간 저장**: 입력 중단 시 자동 저장 및 백엔드 동기화.
   - **데이터 입력**: 마크다운 텍스트, 인라인 이미지 (자동 압축 삽입).

---

## 6. 전제 조건 및 기술적 요구사항

- **Macro 전제**: 최소 1개 이상의 대화 또는 노트가 존재해야 분석이 시작됩니다.
- **Micro 전제**: 분석 대상이 되는 원천 문서(Paper 또는 긴 텍스트)가 시스템에 등록되어 있어야 합니다.
- **업데이트**: 대량 변경 후에는 '그래프 업데이트' 요청이 필요합니다.
